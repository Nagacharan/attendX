# AttendX 🚀

**AttendX** is an AI-powered attendance management system designed to eliminate proxy attendance and streamline the classroom experience. By combining **Face Recognition** (FaceGate) with **Secure, Signed QR Codes**, AttendX ensures 100% integrity in attendance records.

---

## ✨ Key Features

- **FaceGate Identity Verification**: Real-time AI face matching using **DeepFace** to ensure the student's physical presence.
- **Secure AttendX QR**: Time-sensitive, JWT-signed QR codes generated by faculty to prevent tampering or external generation.
- **Dual-Layer Security**: Attendance is only marked if the face verification matches the registered profile AND the QR scan is valid.
- **Live Scan Log**: Faculty can monitor attendance in real-time as students scan the QR.
- **Interactive Dashboards**: Premium "Glassmorphism" UI for both students and faculty.
- **Local Dataset Architecture**: Fast, local processing using JSON/Excel-based data for zero-latency demonstrations.

## 🚀 Getting Started

### **Prerequisites**
- **Node.js** (v18 or higher)
- **Python** (v3.9 or higher)
- **Git**

---

### **1. Backend Setup**

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```

2. Create a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   # For Face Recognition, install DeepFace and OpenCV
   pip install deepface opencv-python
   ```

4. Run the backend server:
   ```bash
   python main.py
   # Or using uvicorn
   uvicorn main:app --reload
   ```

---

### **2. Frontend Setup**

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the development server:
   ```bash
   npm run dev
   ```

## 📸 How it Works

1. **Faculty Session Creation**: The faculty selects a course and generates a unique, signed QR code.
2. **FaceGate Pre-check**: The student opens the AttendX app and undergoes a quick face verification.
3. **QR Scanning**: Once verified, the student scans the faculty's QR code.
4. **Instant Validation**: The backend verifies the JWT signature of the QR and the student's face score, then updates the attendance database instantly.

---

## 📊 Data Management

AttendX currently operates in **Local Dataset Mode** for ease of demonstration:
- **`data_student.json`**: Stores student profiles and attendance percentages.
- **`data_faculty.json`**: Stores faculty details and course assignments.
- **`face_registry/`**: Secure storage for registered student face templates.

> [!TIP]
> The system is architected to be easily migrated to **Supabase** or **PostgreSQL** for large-scale production deployments.

---

## 🛡️ Security

- **JWT Signing**: QR codes are cryptographically signed to prevent spoofing.
- **Expiration Logic**: Tokens expire automatically after the session duration ends.
- **Anti-Proxy**: Real-time face matching prevents students from marking attendance for absent peers.
